Azure Account Setup and Resource Deployment Guide

Objective

The objective of this exercise was to create a Microsoft Azure account, deploy cloud resources, configure cost management, and understand Azure's Shared Responsibility Model.

Step 1: Azure Account Creation

1. Visited the Microsoft Azure Portal.
2. Selected the option to create a free Azure account.
3. Entered personal information.
4. Verified email address.
5. Verified phone number.
6. Completed identity verification.
7. Activated the Azure Free Tier subscription.

Result:

- Azure Subscription 1 was successfully activated.

Step 2: Azure Portal Navigation

After signing in, I explored the Azure Portal dashboard and became familiar with:

- Home Dashboard
- Resource Groups
- Virtual Machines
- Storage Accounts
- Virtual Networks
- Cost Management and Billing
- Subscriptions

This helped me understand how Azure resources are organized and managed.

Step 3: Resource Group Creation

A Resource Group named FirstStep was created.

Procedure:

1. Selected Resource Groups.
2. Clicked Create.
3. Entered Resource Group Name: FirstStep.
4. Selected Region.
5. Reviewed and created the Resource Group.

Purpose:
The Resource Group serves as a logical container for all related Azure resources.

Step 4: Virtual Machine Deployment

A Linux Virtual Machine named firstttt was deployed.

Configuration:

- Operating System: Ubuntu Linux
- Resource Group: FirstStep
- Region: South Africa North
- Public IP Address: Enabled

Deployment Process:

1. Selected Virtual Machines.
2. Clicked Create Virtual Machine.
3. Entered VM name.
4. Selected Ubuntu Linux image.
5. Selected VM size.
6. Configured networking.
7. Reviewed settings.
8. Deployed the Virtual Machine.

Result:
Deployment completed successfully.

Step 5: Cost Management Configuration

Azure Cost Management was configured to monitor cloud spending.

Procedure:

1. Opened Cost Management and Billing.
2. Selected Budgets.
3. Created a monthly budget.
4. Set budget amount to $10.
5. Saved the budget configuration.

Purpose:
The budget helps control cloud spending and prevents unexpected charges.

Step 6: Security and Access Management

Basic Azure security practices were reviewed.

These include:

- Identity and Access Management (IAM)
- User authentication
- Resource permissions
- Secure access to deployed resources

Shared Responsibility Model

Microsoft Azure uses a Shared Responsibility Model.

Microsoft Responsibilities

- Physical data centers
- Hardware maintenance
- Network infrastructure
- Cloud platform availability

Customer Responsibilities

- User accounts
- Access permissions
- Data protection
- Virtual machine configuration
- Application security

Deployment Summary

Subscription: Azure Subscription 1

Resource Group: FirstStep

Virtual Machine: firstttt

Operating System: Ubuntu Linux

Region: South Africa North

Budget Configured: $10 Monthly

Conclusion

The Azure environment was successfully deployed and configured. This exercise demonstrated practical knowledge of Azure account setup, resource management, virtual machine deployment, budgeting, security concepts, and cloud governance.
